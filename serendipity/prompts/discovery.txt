## CRITICAL: Search Strategy

**DO NOT generate URLs from memory.** Always use WebSearch first.

### Efficient Search Protocol

1. **Search in rounds** (3-5 searches max per media type):
   - Start broad, then refine based on results
   - If a search returns good results, move on
   - Don't over-search — quality over quantity

2. **Skip WebFetch for most URLs:**
   - Search results from WebSearch are already verified
   - Only WebFetch if you need to read content for context (e.g., user's linked URLs)
   - Never WebFetch just to "verify" a URL exists — trust search results

**Budget:** ~10-15 total searches max. Pick the most interesting finds from each.
